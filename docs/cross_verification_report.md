# 교차검증 상세 리포트

작성일: 2026-06-29

이 문서는 `Korean Bible 1961 KRV-Hanja` 공개 초안의 검수 기준, 진행
절차, 산출물, 한계 표현을 정리한 상세 리포트입니다. 공개 README에는
요약만 두고, 세부 검수 계보는 이 문서에서 추적하도록 분리했습니다.

## 검수 목적

국한문 성경 데이터가 개역한글 1961 국문 기준과 구조적으로 대응하는지
확인하고, 한자 병기와 국한문 표기는 공개 대조 자료를 참고하여 보존
가능한 형태로 정리하는 것을 목표로 했습니다.

검수의 중심 원칙은 다음과 같습니다.

- 국문 기준은 개역한글 1961로 둡니다.
- 국한문 표기는 원본 국한문 자료를 보존 대상으로 삼되, 공개 대조 자료와
  비교해 후보를 분류합니다.
- 자동 검수로 발견한 차이는 사람의 승인 또는 공개 표기 정책을 거친
  항목만 후보본에 반영합니다.
- 공식 국한문 정본과 완전 동일하다고 단정하지 않습니다.

## 기준 자료

### 대한성서공회 개역한글판 1961

- 용도: 국문 본문 기준 확인
- 역본 정보: <https://www.bskorea.or.kr/prog/trans_feature2.php>
- 성경읽기: <https://www.bskorea.or.kr/bible/korbibReadpage.php>

대한성서공회 자료는 개역한글 1961 국문 기준을 확인하기 위해 사용했습니다.
국한문 한자 병기의 공식 대조본으로 사용한 것은 아닙니다.

### 위키문헌 개역간이국한문한글판

- 용도: 국한문/한자 병기 공개 대조 참고
- 링크: <https://ko.wikisource.org/wiki/개역간이국한문한글판>

위키문헌 자료는 한자 병기, 국한문 문구, 일부 누락 또는 이형 후보를
검토하기 위한 공개 참고 자료로 사용했습니다. 이 자료를 공식 국한문
정본으로 단정하지 않습니다.

## 진행 절차

1. 원본 보존
   - 프로젝트의 국한문 원본 TXT를 보존하고 해시 기록을 생성했습니다.
   - raw JSON을 생성하여 원본 파싱 결과와 이후 후보본을 분리했습니다.

2. 구조 검수
   - 66권/1,189장 구조를 확인했습니다.
   - 권별 JSON과 전체 통합 JSON의 구조가 동일하게 재현되는지 확인했습니다.

3. 국문 기준 한글 환원 대조
   - 국한문 본문에서 한자를 제거하거나 환원한 `hangulText`를 개역한글
     1961 국문 기준과 대조했습니다.
   - 초기 구조/국문 기준 대조에서 7,762건의 차이를 분류 대상으로
     기록했습니다.

4. 외부 공개 자료 대조
   - 위키문헌 개역간이국한문한글판을 공개 참고 자료로 삼아 국한문 표기
     후보와 누락 후보를 비교했습니다.
   - 외부 대조 기록 93건을 생성하고, 각 항목을 보존 후보, 보정 후보,
     공개 표기 정책 후보로 분류했습니다.

5. 승인 기반 보정
   - primary 보정 22건을 승인 후 적용했습니다.
   - supplemental 보정 12건을 승인 후 적용했습니다.
   - 에베소서 3:1, 예레미야 32:3 등 추가 확인 항목은 개역한글 국문 기준
     결정을 반영했습니다.

6. 공개 표기 정책 결정
   - 공개 표기 정책 대상 66건을 검토했습니다.
   - `publication_policy_decision_template.csv` 기준으로 66건 승인,
     0건 거절 상태를 기록했습니다.

7. 최종 후보본 재감사
   - 승인 manifest만 반영한 round2 후보본을 생성했습니다.
   - 최종 후보본 전체를 다시 감사하고 공개 패키지를 생성했습니다.

## 공개 표기 원칙

공개 문서에서는 다음과 같이 표현합니다.

- “개역한글 1961 국문 기준으로 검수”
- “대한성서공회 자료는 국문 기준 확인에 사용”
- “위키문헌 개역간이국한문한글판은 국한문 대조 참고 자료로 사용”
- “공식 국한문 정본과 완전 동일하다고 단정하지 않음”

다음 표현은 공개 출처 문구에서 사용하지 않습니다.

- 비공개 개인 확인 자료를 공개 출처로 삼았다는 표현
- 문서화되지 않은 개인 검토를 공개 출처로 삼았다는 표현
- 공개 참고 자료의 권위를 과장하는 표현

## 한계와 후속 과제

- 공개 자료만으로 공식 국한문 정본과 완전 동일성을 보증하기는 어렵습니다.
- 한자 병기는 원본 자료와 공개 대조 자료의 차이를 보존 정책으로 관리해야
  합니다.

  이 리포트를 갱신하는 것이 좋습니다.
- GitHub 공개 전에는 저작권과 저장소 설명 문구를 프로젝트 관리자가 최종
  확인하는 절차를 권장합니다.
